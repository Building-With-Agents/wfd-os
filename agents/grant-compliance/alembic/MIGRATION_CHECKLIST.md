# Migration review checklist

Before applying any Alembic migration — autogenerated OR hand-written — verify:

1. **Read the entire migration file**, both `upgrade()` and `downgrade()`, before running `alembic upgrade`.
2. **Every operation is scoped to `schema='grant_compliance'`.** No operations on `public` or any other schema.
3. **No DROP statements in `upgrade()`** unless the drop is intentional, reviewed, and the dropped table is definitively part of `grant_compliance`.
4. **Table and column names match what you expect** — autogenerate sometimes produces surprises for naming conventions or ordering.
5. **Run `alembic upgrade head --sql` first** to see the exact SQL before applying it live, especially in production.

If any check fails, stop. Do not apply. Fix the root cause (usually `env.py` configuration) and regenerate.

This checklist exists because of a near-miss during Step 0 where autogenerate attempted to DROP all of wfd-os's `public.*` tables. See CLAUDE.md "Enforced constraints."
